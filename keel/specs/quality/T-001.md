# T-001 — Structured End-Of-Run Assessment

## Goal

Replace the current shallow end-of-run summary with a structured assessment artifact and richer Keel note payload.

## Why

Today Cloudy can succeed and still leave weak truth behind. A one-line summary is not enough to evaluate run quality.

## Scope

- define an assessment shape
- write `assessment.json` into the run directory
- include summary, checks, artifacts, risks, and next action
- post a structured summary into Keel notes

## Acceptance Criteria

- after a run, `.cloudy/runs/<run>/assessment.json` exists
- the assessment includes:
  - summary
  - tasksDone/tasksFailed
  - checksPassed/checksFailed
  - artifactsProduced
  - qualityVerdict
  - risks
  - recommendedNextAction
- Keel note write-back contains more than a one-line sentence and clearly exposes the same assessment fields
- existing successful run behavior does not regress

## Non-Goals

- changing Keel task status semantics yet
- adding watchdog logic
