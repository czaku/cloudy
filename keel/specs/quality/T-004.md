# T-004 — Review-Ready Keel Write-Back

## Goal

Stop treating every successful Cloudy run as a fully done Keel task.

## Scope

- introduce a review-ready or equivalent non-final state in the write-back contract
- distinguish execution success from accepted completion
- only close to done when proof and acceptance gates are satisfied

## Acceptance Criteria

- successful runs can be written back without always closing tasks to done
- blocked or failed runs still write useful decision context
- the final state written to Keel reflects acceptance quality, not only command success
