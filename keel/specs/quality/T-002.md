# T-002 — Proof Contract And Artifact Enforcement

## Goal

Make proof requirements first-class so Cloudy can fail runs that claim success without evidence.

## Scope

- define required proof fields per task
- support required artifacts, screenshots, commands, and doc updates
- fail close-out when declared proof is missing

## Acceptance Criteria

- tasks can declare required proof fields
- validator fails when required proof is missing
- run output clearly names missing proof items
- proof checks are separated from generic execution success
