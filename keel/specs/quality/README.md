# Cloudy Quality Program

This folder defines the concrete quality-improvement program for `cloudy`.

The goal is not just better code. It is better delivery truth.

Cloudy should be able to answer, reliably:

- what changed
- what proof exists
- whether acceptance actually passed
- whether docs and Keel agree
- whether the run is green, yellow, or red

## Active Sequence

1. `T-001` structured run assessment
2. `T-002` proof contract
3. `T-003` acceptance reconciliation
4. `T-004` review-ready Keel write-back
5. `T-005` lane collision checks
6. `T-006` watchdog loop
7. `T-007` preflight baseline separation
8. `T-008` operator-facing quality views
